 به طور کلی، روال اصلی این است:

مخزن محلی خود را بادستورات آپدیت میکند
git clone... or git pull

سپس فایل ها را به صورت محلی ویرایش میکند با دستورات:
git add...

git commit...

git push...

git clone:
برای کپی کردن یک مخزن از GitHub، از git clone استفاده کنید. به مخزن GitHub خود بروید، روی دکمه سبز رنگ "Clone or download" کلیک کنید و URL را کپی کنید. در ترمینال خود، به مکان مورد نظر خود بروید git clone را تایپ کرده و در URL خود Paste کنید.در واقع Clone یک نسخه کامل از مخزن را در دستگاه محلی شما دانلود می کند و منشا آن را در GitHub ثبت می کند.این تضمین می کند که یک اتصال وجود دارد، بنابراین می توانید تغییرات خود را به GitHub فشار دهید یا به روز رسانی ها را واکشی کنید. کلون شما تاریخچه کامل را در پوشه پنهان .git ذخیره می کند.

git status:
وضعیت Git دوست شماست. آن را اغلب تایپ کنید!

وضعیت git

با استفاده از یک ویرایشگر متن یا خط فرمان یک فایل جدید در مخزن خود ایجاد کنید:

echo "چند یادداشت جالب" > notes.txt

سپس دوباره وضعیت را بررسی کنید!
git add:
هنگامی که تغییراتی را در مخزن خود ایجاد کردید، باید به Git بگویید تا ردیابی فایل را شروع کند. تابع git add فایل ها را به "منطقه مرحله بندی" اضافه می کند، به این معنی که Git اکنون تغییرات را دنبال می کند.
git add notes.txt

git status

git commit:
ما یک فایل را تغییر دادیم، آن را به قسمت مرحله‌بندی اضافه کردیم، و اکنون آماده هستیم که یک عکس فوری بگیریم و این تغییرات را برای همیشه ذخیره کنیم! بیایید متعهد شویم!

هر commit یک عکس فوری از وضعیت مخزن کامل به همراه نام، زمان و پیام committer را ثبت می کند. پیام لازم است! این یادداشت شما برای آیندگان است تا بتوانید به خاطر بیاورید که چرا در آینده تغییراتی ایجاد کرده اید.
git commit -m "add notes file"

git status

ما یک نسخه جدید از اسناد را در مخزن ذخیره کرده ایم و می توانیم در این تاریخچه پیمایش کنیم. به جای تعهدات بزرگ، بسیاری از تعهدات متمرکز کوچکتر را انجام دهید، زیرا بازگرداندن، بررسی و ادغام تغییرات کوچک آسان تر خواهد بود.
git push:
توجه داشته باشید که وضعیت git اکنون می گوید "شاخه شما جلوتر از مبدا/مستر است". Origin/Master روش استاندارد برای نامگذاری شاخه اصلی مخزن راه دور است. ما تغییرات خود را در مخزن به صورت محلی به پایان رساندیم، اکنون باید آنها را با استفاده از فشار به نسخه میزبانی شده در GitHub اضافه کنیم.
git push origin master
